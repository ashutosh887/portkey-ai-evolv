# Evolv Web - Log Generator

Next.js application for generating prompt logs that are sent to Portkey observability and can be ingested by the Evolv system.

## Features

- Generate individual prompt logs
- Auto-generate logs at configurable rates (up to 5 logs/second)
- Real-time display of recent logs (latest 10)
- Integration with Portkey SDK
- Clean, modern UI

## Setup

### Prerequisites

- Node.js 18+ and npm/yarn/pnpm

### Installation

```bash
cd apps/web
npm install
```

### Environment Variables

Copy `.env.local.example` to `.env.local` and configure:

```bash
NEXT_PUBLIC_PORTKEY_API_KEY=your_portkey_api_key
NEXT_PUBLIC_PORTKEY_PROVIDER=your_provider_id
NEXT_PUBLIC_MOCK_MODE=false
```

Note: `NEXT_PUBLIC_PORTKEY_PROVIDER` is optional. If you have a provider configured in your Portkey dashboard, you can reference it here. Otherwise, Portkey will use your default gateway configuration.

### API Configuration

The web app connects to the Evolv API backend. Configure the API URL:

```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

If not set, defaults to `http://localhost:8000`. Make sure the API server is running when using the dashboard, families, or prompts pages.

### Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

### Build

```bash
npm run build
npm start
```

## Usage

1. **Manual Generation**: Enter a prompt, select a model, and click "Generate Log"
2. **Auto Generation**: Set the rate (logs per second) and click "Start Auto Generation"
3. **View Logs**: Recent logs appear in the right panel (latest 10)

## Integration with Evolv

Logs generated by this application are automatically sent to Portkey observability. The Evolv system can then:

1. Ingest logs from Portkey API
2. Extract prompt DNA
3. Cluster into families
4. Track evolution

This creates a complete pipeline: **Generate → Observe → Analyze → Evolve**
